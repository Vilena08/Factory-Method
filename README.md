# Factory-Method
Фабричный метод — это порождающий паттерн проектирования, который определяет общий интерфейс для создания объектов в суперклассе, позволяя подклассам изменять тип создаваемых объектов.
(из книги "Погружение в паттерны проектирования")

Обладает следующими характеристиками:
Тип:  порождающий

Принцип работы:

1. Клиентский код:

   - Создает объект фабрики.

   - Вызывает метод фабричного метода, чтобы создать нужный объект.

2. Фабрика:

   - Проверяет входные параметры и выбирает соответствующий подкласс.

   - Создает и возвращает объект подкласса.

3. Подклассы:

   - Реализуют метод фабричного метода, создавая объекты конкретных классов.


Назначение:  Создавать отдельные объекты, не указывая их конкретные классы.

Инкапсулировать (скрывать от посторонних глаз) логику создания объекта, позволяя подклассам определять, какой объект будет создан.

 Унифицировать создание объектов, обеспечивая единый интерфейс для разных вариантов создания.
Метафора: Фабрика может быть похожа на ресторан, где повара (подклассы) создают различные блюда (объекты) на основе заказов клиентов (клиентского кода).
Потенциальные проблемы: Множество классов, усложнение тестирования.

Отличия от Abstract Factory:

Factory Method про создание элементов волков-одиночек, тогда как Abstract Factory спавнит сразу  семейство объектов, связанных меж собой и которые используются вместе, не порознь.
